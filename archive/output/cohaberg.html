18.4 Static netdx

See [SHAMP/egonet/analyses/ergms_final/netdx/cohab_static_b1i1e6.html/Rmd]. Some notes about using this script:

  • You should go through the Rmd script line by line, and then look at the corresponding html output, to understand what it does. I don’t think there is a fast-track way to circumvent this step.

  • It knits the results into an HTML file as well as saves key output to be read in by this GitBook. If you want to do a new netdx run without overwriting prior results, copy the Rmd file using a new name. In the new file, edit the following file names on the following lines so that they match your new file name in some way:

    • The .R file name on line 36
    • The .txt file name suffix on lines 248 and 249
    • The .png file name suffix on line 258

Issue #134 summarizes some problems encountered and corresponding solutions in the first post. The TOC links to in-depth investigations of the internals of static and dynamic netdx calls. Two important takeaways:

  1. The default starting net for netdx simulations is the empty network.
  2. Using sequential=TRUE is like running a dynamic sim in that each sim depends on the previous one, sort of like a time step. However there is no dissolution.

In the new file, edit the following file names on the following lines so that they match your new file name in some way:

Issue #134 summarizes some problems encountered and corresponding solutions in the first post. The TOC links to in-depth investigations of the internals of static and dynamic netdx calls. There were two important takeways:

  1. The default starting net for the netdx simulations was the empty network. We used ee.netest to use the fitted net instead.
  2. The MCMC.burnin parameter essentially controls the MCMC interval for dynamic sims, since it is a “burnin per time step.”

This post (as part of Issue #134) gives the overview of netdx vs ee.netdx - we used the latter function to achieve #1.

Issue #146 summarizes both homogeneous and heterogeneous dissolution diagnostics pretty thoroughly. For homogeneous dissolution, diagonstics look good except for nodematched terms in the cohab model, which plateau too low. This is noted in this #134 post as well.

Nodematch terms plateau low. We opened Issue #149 to track progress. We are revisiting in #193.
