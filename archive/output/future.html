5.8 Simulation run limitations

There was a twofold limitation in running the simulations, and will be passed on for future work:

  1. the High Performance Cluster schedulers (HYAK) on the UW servers were upgraded in the past year to the SLURM scheduler. Due to these changes, there have a number of modifications that were made. This will require more time to properly navigate the newer generation of Hyak.

  2. To incorporate Approximate Bayesian Computations (ABC) to the simulation runs, we rely on the EpiABC package. This package outputs parameters that are expected to converge and approximate the expected prevalence, but there have been several problems in the package. We will manually investigate paramter spaces to approximate prevalence, and will use ABC for future work. See Issues #89 for ABC targets, #171 ABC priors, and #90 ABC levers.

Microbenching the project would also showcase areas that are holding back the process, as shown in Issue #154, and this will be pursued for future work.

5.9 Dynamic EpiModel Parameter application

The current workflow for applying modelling parameters to the simulations is by hardcoding the values into the code. Some work has been made to automate this process by recording the values and the source of the parameter data into an Excel sheet. This work is recorded under https://github.com/statnet/EpiModelHIV/blob/shamp_params/data/Model%20Parameters.Sara.work.xlsx and is under the shamp_params branch in EpiModelHIV. The parameters have been added to the statnet/EpiModelHIV/R/params.shamp.R file.

Future work will be to test the entire EpiModelHIV with the parameters included in new params.shamp.R code.
